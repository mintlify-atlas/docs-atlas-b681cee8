# Requirements

Before running the BD-JB exploit, make sure you have everything you need. This page covers hardware requirements, firmware compatibility, system settings, and necessary tools.

---

## Firmware Compatibility

BD-JB supports PlayStation 4 consoles running firmware versions **9.00 through 12.50**.

<Note>
  **Important**: The Lapse exploit (automatic USB payload loading) only works on firmware 9.00-12.02. Firmware 12.50 requires using RemoteJarLoader with custom JAR payloads.
</Note>

| Firmware Version | Lapse.iso Support | RemoteJarLoader.iso Support |
|------------------|-------------------|--------------------------|
| 9.00 - 12.02 | ✅ Yes | ✅ Yes |
| 12.50 | ❌ No | ✅ Yes |
| 12.52+ | ❌ No | ❌ No |

<Warning>
  BD-JB does **not** work on PlayStation 5. This exploit is exclusively for PS4 consoles.
</Warning>

### How to Check Your Firmware Version

On your PS4:
1. Go to **Settings → System → System Information**
2. Look for **System Software** version
3. Verify it's between 9.00 and 12.50

    - A Blu-ray disc burner (external or internal)
    - Compatible with your computer (Windows, Linux, or macOS)

    <Note>

  </Accordion>
  
  <Accordion title="USB Drive (for Lapse.iso)">

    <Note>

    </Note>

## Software Requirements

### Disc Burning Software

Choose the appropriate tool for your operating system:

| Operating System | Recommended Tool | Download Link |
|------------------|------------------|---------------|
| **Windows** | ImgBurn | [imgburn.com](https://www.imgburn.com) |
| **Linux** | K3b | [apps.kde.org/k3b](https://apps.kde.org/k3b) |
| **macOS** | BD-SimpleBurn | [github.com/C4ndyF1sh/BD-SimpleBurn](https://github.com/C4ndyF1sh/BD-SimpleBurn) |

<Warning>
  Make sure to burn the ISO as a **disc image**, not as a data file. The disc must be bootable.
</Warning>

### Python (for sending payloads)

If using **RemoteJarLoader** or sending additional payloads over the network:

- Python 3.x installed on your computer
- Included `payload_sender.py` script from the BD-JB release

Example usage:
```bash
python payload_sender.py Lapse.jar 192.168.1.100 9025
```

### Homebrew Payload

You'll need a homebrew enabler payload such as:

- **GoldHEN** (recommended)
- **ps4-hen**

Rename your chosen payload to `payload.bin` and place it on your USB drive (for Lapse.iso) or send it via network after the exploit runs.

## PS4 System Settings

Before running the exploit, configure these settings on your PS4:

<Steps>
  <Step title="Enable HDCP">
    1. Go to **Settings → System → HDCP**
    2. **Enable HDCP**
    
    <Note>
      HDCP is required for Blu-ray playback on PS4.
    </Note>
  </Step>
  
  <Step title="Disable Automatic Updates">
    1. Go to **Settings → System → Automatic Downloads and Updates**
    2. **Uncheck all automatic update options**
    
    <Warning>
      This is critical! An automatic firmware update will patch the exploit and prevent BD-JB from working.
    </Warning>
  </Step>
  
  <Step title="Configure Notification Settings">
    1. Go to **Settings → Notifications → Pop-Up Notifications**
    2. **Turn off "Disable Pop-ups while playing video"**
    
    <Note>
      This allows you to see notifications from the exploit, confirming it's running successfully.
    </Note>
  </Step>
  
  <Step title="Activate Blu-ray Playback (if needed)">
    If you've never used a Blu-ray disc on your PS4:
    
    1. **Disable automatic updates first** (see step 2 above)
    2. Connect your PS4 to the internet
    3. Insert any commercial Blu-ray disc to activate playback
    4. Disconnect from the internet after activation
    
    <Warning>
      Only connect to the internet if Blu-ray playback is not yet activated. Make sure automatic updates are disabled before connecting!
    </Warning>
  </Step>
</Steps>

## Network Configuration

### For RemoteJarLoader

If using **RemoteJarLoader.iso**, ensure:

- PS4 is connected to your local network (Wi-Fi or Ethernet)
- PS4 and your computer are on the **same network**
- Port **9025** is accessible (not blocked by firewall)
- You know your PS4's IP address

#### Finding Your PS4's IP Address

1. Go to **Settings → Network → View Connection Status**
2. Note the **IP Address** (e.g., 192.168.1.100)

### For BinLoader (after exploit runs)

After the exploit runs successfully:

- Port **9020** will be open for additional payloads
- Use `payload_sender.py` to send `.bin` files:

```bash
python payload_sender.py payload.bin 192.168.1.100 9020
```

### For Logging (Optional)

If you want to view real-time logs from the exploit:

- Port **18194** must be accessible on your computer
- Run `log_client.py` before inserting the disc:

```bash
python log_client.py
```
